### Cardinality of the Power Set of a Set
For Sn = {a<sub>1</sub>, a<sub>2</sub>,..., an}, what recurrence relation does |pow(S<sub>n</sub>)| satisfy?
S1 = {ø, {a1}}, S2 = {ø, {a<sub>1</sub>}, {a<sub>2</sub>}, {a<sub>1</sub>, a<sub>2</sub>}} ......
So S<sub>n+1</sub> contains all elements already in S<sub>n</sub>, together with the unions of those elements with a<sub>n+1</sub>. (Here Se denotes the individual elements of S.) $$S_{n+1}=\{{Se_n, Se_n∪\{a_{n+1}\}}\}$$
Therefore |pow(S<sub>n+1</sub>)|  = 2|pow(S<sub>n</sub>)|

### Binary Strings Without “111”
What recurrence counts the number of n-bit binary strings B<sub>n</sub> that do **not** contain the substring “111”?
B<sub>1</sub>：{1, 0} →2
B<sub>2</sub>：{10, 01, 11, 00} →4
B<sub>3</sub>：{110, 000, 010, 011, 001, 100, 101} →7
B<sub>4</sub>：{1100,  0011, 1011, 1101, 0000, 1000, 0001, 0101, 1010, 0100, 0110, 0010, 1001} →13
......
Observe the elements of B<sub>4</sub>:

- {0110, 0000, 0010, 0011, 0001, 0100, 0101} come from B<sub>3</sub>, and all of them can be obtained by **adding a leading 0**.
  {1010, 1001, 1011, 1000} come from B<sub>2</sub>, and all of them can be obtained by **adding the leading bits 10**.
  {1101, 1100} come from B<sub>1</sub>, and all of them can be obtained by **adding the leading bits 110**.
- Equivalently, one may say: “every element of **B<sub>3</sub>** may have a leading 0 added, while those whose first two bits are not 11 may also have a leading 1 added.”
Hence |B<sub>n</sub>| = |B<sub>n-1</sub>| + |B<sub>n-2</sub>| + |B<sub>n-3</sub>| for n>3.

### Towers of Hanoi
There are three pegs, and n disks stacked on the leftmost peg from largest to smallest. Under the rule that on every peg a larger disk must always remain below a smaller one, how many moves f(n) are needed to reproduce the entire stack on the rightmost peg without changing the order?

- For brevity, let the smallest disk be 1 and the largest be n; name the pegs a, b, and c.

- n=1: 1->c

- n=2: 1->b、2->c、1->c

- n=3: 1->c、2->b、1->b、3->c、1->a、2->c、1->c

  ![](https://i.postimg.cc/pr1hBVg7/hanota-divide-and-conquer.png)

Looking at n=3, the idea is simple: first move the two smaller disks away, then move the largest disk to the rightmost peg, and finally move the two smaller disks onto it. The middle step is equivalent to n=1, while the first and last parts both amount to moving two disks, in the same relative order, from one peg to another (“from a to b” and “from b to c”). Since the three pegs have no intrinsic individuality, the number of steps depends only on the number of disks, not on which peg is the source or the destination. Thus, analyzing f(3) amounts to calling f(2), then f(1), then f(2).
**So we may conjecture that f(3) = 2f(2)+1, and in general f(n) = 2f(n-1)+1**, as illustrated below.

![](https://i.postimg.cc/GtpTYTSG/hanota-recursive-tree.png)

Following the same logic, every Towers of Hanoi problem can be solved recursively by a **three-part decomposition**, until all that remains are individual moves. Therefore, in Python we essentially only need a **single-move routine, a three-way recursive routine, an entry routine, and the main routine**:

```python
def move(src: list[int], tar: list[int], name_src: str, name_tar: str, steps: list[int], A, B, C):
    # 移动一个圆盘，并打印当前状态
    pan = src.pop() # 记录并去掉 source 数组顶部的元素
    tar.append(pan) # 将其加到 target 数组
    steps[0] += 1
    print(f"第 {steps[0]} 步: {name_src} -> {name_tar}")
    print(f"A={A}, B={B}, C={C}
")

def dfs(i: int, src: list[int], buf: list[int], tar: list[int], # 递归 f(i)
        name_src: str, name_buf: str, name_tar: str,
        steps: list[int], A, B, C):
    if i == 1:
        move(src, tar, name_src, name_tar, steps, A, B, C)
        return
    dfs(i - 1, src, tar, buf, name_src, name_tar, name_buf, steps, A, B, C)  # f(i-1)
    move(src, tar, name_src, name_tar, steps, A, B, C)  # f(1)
    dfs(i - 1, buf, src, tar, name_buf, name_src, name_tar, steps, A, B, C)  # f(i-1)

def solve_hanota(A: list[int], B: list[int], C: list[int]):
    # 求解并返回总步数
    n = len(A)
    steps = [0]
    dfs(n, A, B, C, "A", "B", "C", steps, A, B, C)
    return steps[0]

if __name__ == "__main__":
    num = int(input("请输入汉诺塔圆盘数量："))  # 注意 input 自动存储为 type:str
    A = list(range(num, 0, -1))  # [num, num-1, ..., 1]
    B = []  # 也可以写 list[int]
    C = []
    total_steps = solve_hanota(A, B, C)
    print(f"共移动了 {total_steps} 步。")
```

## Solving Recurrence Relations

### Closed forms for first-order linear recurrences
- Homogeneous linear recurrence (coefficient of the previous term ≠ 1): $$a_k=r a_{k-1}\ 	o\ a_k=r^{k-1}a_1$$
- Nonhomogeneous baseline linear recurrence (coefficient of the previous term = 1): $$a_k = a_{k-1} + b\ 	o\ a_k = a_1 + (k-1)b$$
- Nonhomogeneous linear recurrence with coefficient ≠ 1, such as Hanoi: $$a_k=r a_{k-1}+b\ 	o \ a_k=r^{k-1}a_1+rac{b\,(r^{\,k-1}-1)}{r-1}$$

### Generating-function method

For a first-order recurrence defining a sequence a, with a general form such as $$a_{n} = c_{n-1}·a_{n-1} + c_{n-2}·a_{n-2},\ n≥2$$

one may assume the generating function is $$F(x) = \sum_{n=2}^{\infty} a_nx^n$$

Then the recurrence can be written in generating-function form as $$F(x)-a_0-a_1x=c_{n-1}x·(F(X)-a_0)+c_{n-2}x^2·F(x)$$

Hence one obtains $$F(x)={{a_0+a_1x-a_0c_{n-1}x}\over{1-c_{n-1}x-c_{n-2}x^2}}$$

Factor the quadratic denominator, decompose it into the sum of two fractions, and determine their numerators by matching coefficients with the original numerator.

Then recover the generating function itself using the identity $$raceta{1-lpha x}=\sum_{n=0}^{\infty}eta·lpha^n x^n$$

Finally, once the generating function has been expanded, one can read off the closed form of the sequence by removing the summation symbol and the factor x<sup>n</sup>.
