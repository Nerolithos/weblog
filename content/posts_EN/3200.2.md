# CSC3200 LEC 3-4 Arrays, Pointers, and Streams

<br>

## Arrays and Pointers

Arrays and pointers are two fundamentally different data types, but they are closely related, and parts of their syntax are mutually compatible.

<br>

### C-Style Character Arrays

- Recall that `a.c_str()` converts string `a` into a `const char*`. Its effect is essentially the same as `&a[0]`.
- For string length, use the member methods `.size()` or `.length()`. For the length of a character array accessed through a pointer, use `strlen()`.
- As a pointer, whether or not it is `const`, you can perform **offset operations** such as `s.c_str()[n] ≡ s.c_str()+n`, and also **pointer-difference calculations** such as `s.c_str()[5]-s.c_str[0] == 5`.
- This is a classic example of the compatibility between pointer syntax and array syntax.

```cpp
string s = "hello";
s.length();  // 5
cout << boolalpha << (strlen(s.c_str()) == (s.c_str[4]-s.c_str[0])) << '\n';  // True
cout << toupper(s.c_str()[0]);  // 'H'(`toupper` only handles characters)
```

- **Note carefully:** for a string `x`, the type of `&x` is `string*`, whereas `x.c_str()` yields `const char*`. These two types cannot be assigned to each other directly. The following declarations are compatible with the `c_str()`-style representation:

```cpp
char* arr = new char[100];  // allocated on the heap; must be freed manually with `delete[] arr;`
char arr[100];  // allocated on the stack; released automatically
```

<br>

### Definitions of `strlen`

```cpp
int strlen1(char* s){ int n=0; while(s[n] != '\0') n++; return n; }
int strlen2(char* s){ int n=0; while(*s++ != '\0') n++; return n; }
int strlen3(char* s){ char* p=s; while(*p!='\0') ++p; return p - s; } 
```

1. Here `n` serves both as an index into the character-pointer array and as a counter, increasing until it reaches `\0`.

2. Here `n` is only a counter, while `*(s++)` advances the pointer until it points to `\0`.

3. Here no counter is used; instead, the code directly computes the pointer offset between the start and end positions.

- Pointer increment `*p++` is equivalent to `*(p++)`: “return the value pointed to by the pointer, then increment the pointer.”

```cpp
string a = "hello";
const char* p = a.c_str();   // or `char* p = &a[0];`
while (*p) {
    cout << *p << '\n';
    p = p + 1;
}
```

- The code above is equivalent to the following:

```cpp
while (*p) {
    cout << *p++ << '\n';   // output the current character while advancing the pointer
}
```

<br>

### A Pointer “As” an Array

- If a pointer is made to a string buffer, it points at the first character, which is `ptr[0]`.
- A pointer can be indexed like an array, so here `*(ptr) ≡ ptr[0] ≡ 'h'`.
- As noted earlier, `s.c_str()` is closely related to `&s[0]`, both behaving like pointers to the first character.

```cpp
string s = "hello";
const char* ptr = s.c_str();  // *(ptr+2) == 'l'
```

<br>

### An Array “As” a Pointer

- In expression context, an array name implicitly **decays** into a pointer to its first element, such as `&cstr[0]`.
- The fixed array version does not require manual destruction, whereas the dynamically allocated version does.

```cpp
class StringA {
    char cstr[256];   // fixed array: gives the object a fixed-size stack allocation, with no extra dynamic allocation needed
    char* ptr = cstr;  // legal
};
class StringB {
    char* ptr = new char[256];  // dynamically allocate heap space; memory must later be managed manually
};
```
<br>

### Arrays Are Not Equivalent to Pointers

- But **arrays are not pointers**. An array name is a non-modifiable lvalue: the decayed array may be assigned to a pointer, but a pointer cannot be assigned back to the array.
- Note that `arr` decays to a pointer to its first element, while **`&arr` has the type “pointer to the entire array.”**
- If `double arr[5]; double* dp = arr;`, then the identity `arr[i] ≡ *(arr+i) ≡ *(dp+i) ≡ dp[i]` holds.

```cpp
int arr[4] = {10, 20, 30, 40};
int* ptr = arr;        // legal: the array name decays into `int*`    (1)
int  x = arr[2];       // legal, equivalent to *(arr + 2) or *(ptr + 2)
sizeof(arr);           // 16 (the total bytes of 4 ints)
sizeof(ptr);           // 8 (on a 64-bit machine, pointers are usually 8 bytes)
arr = ptr;             // ❌ illegal: an array is not an assignable object    (2)
```

- Small note: an **lvalue** is something that may appear on the left-hand side of `=`, such as `*ptr` or a named variable; an **rvalue** is usually a temporary or non-assignable expression.

## The C Standard-Library Copy Function `strncpy`

<br>

### Its Use in Implementing a String Class

```cpp
char* strncpy(char* destination, const char* source, std::size_t num);
std::strncpy(this->c_str + this->strlen, other.c_str, copylength);
```
- `strncpy` copies at most `num` characters from `source` to `destination`. If `source` is shorter than `num`, the rest is padded with `\0`.
- **If `source` has length ≥ `num`, then only the first `num` characters are copied, and no terminating `\0` is appended automatically.**
- In the example below, another character array is appended to the current one. (The `this->` qualifiers could also be omitted.)

A full example is a C++ `String` class defining `operator+=` to concatenate two strings `str1` and `str2`:
```cpp
void string::strcat(string const& other) {  // "cat" whether the append operation will exceed MAXLEN 256
	int total = strlen + other.strlen;
	if (total >= MAXLEN) total = MAXLEN - 1;  // exceeding 255, remaining char WILL be discarded
	int copylen = total - strlen;
	if (copylen > 0) {
		std::strncpy(this->c_str + strlen, other.c_str, copylen);
		strlen = total;
		c_str[strlen] = '\0';  // in case of exceeding
	}
}
// ...... //
string& string::operator+=(string const& other) { strcat(other); return *this; }
```
- This code appends instance `other` to the end of the current string, truncates any overflow beyond capacity, and guarantees a final `\0`, so the routine remains safe for repeated use.
```cpp
str::cout << string(str1) + string(str2) + string(str3) << '\n';
```

<br>

### Implementing `strncpy` with Pointer Increments

```cpp
#include <iostream>
using namespace std;

void str_cpy(char* dst, const char* src) {
    while (*dst++ = *src++);
}

int main() {
    char ptr1[] = "Hello";
    char ptr2[100];   // buffered (the stack buffer must be large enough)
    str_cpy(ptr2, ptr1);
    cout << ptr2 << '\n';  // output Hello
    return 0;
}
```

- Note that the `while` condition does **not** use `==`. The condition performs an assignment, whose value is the character just written into `dst[]`. Any nonzero character counts as true, and the loop stops only when `\0` is reached.

### Keeping Decimal Places in Output
- CPP style: setprecision()
```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double d = 3.141592653589793;
    cout << "default + setprecision(2): " << setprecision(2) << d << '\n'; // 3.1 (two significant digits)
    cout << fixed << setprecision(2);
    cout << "fixed + setprecision(2):   " << d << '\n';  // 3.14
    cout << scientific << setprecision(2);
    cout << "scientific + prec(2):     " << d << '\n';  // 3.14e+00
    cout << defaultfloat; // restore the default mode
}
```
- C style: printf()
```cpp
#include <iostream>
#include <cstdio>
using namespace std;
double db = 3.14159265358979323846;
printf("This is a double: %.2f\n", db);  // roughly equivalent to `fixed`
```
(`%d` for integers, `%c` for characters, `%s` for strings)

## FileStream

Used for outputting to files or reading information from them.

- `ifstream`: defaults to `std::ios::in` (read-only mode)
- `ofstream`: defaults to `std::ios::out | std::ios::trunc` (overwrite mode)
- `.open(cchar_file, mode)`: opens a file with the specified mode
- `.get()`, `.put()`: read and write one character
`<fstream>` does not work like Python-style chained file objects; one must explicitly open and use the streams.

Copying a file by streams:
```cpp
int main() {
    ifstream fin;
    ofstream fout;
    fin.open("infile.txt"); // ”infile.txt".c_str()
    fout.open("outfile.txt", std::ios::out | std::ios::app); // append mode
    char ch;
    while (fin.get(ch)) {   // read one character at a time; stop on failure / EOF
        fout.put(ch);
    }
    fin.close();
    fout.flush();
    fout.close();
    return 0;
}

```

## Memory Allocation

- Ordinary local variables created without `new` live on the stack and are destroyed automatically when the function returns.
- All objects allocated with `new` live on the heap and must later be released manually using `delete` or `delete[]`, otherwise memory leaks occur. In short, every `new` should conceptually correspond to a `delete`.

```cpp
int* indexArr(int n) {
    int* arr = new int[n];
    for (int i = 0; i < n; i++) arr[i] = i;
    return arr;
}
int main() {
    int* digits = indexArr(10);
    delete[] digits;  // must be released
    return 0;
}
```

## `const` and `static`

- Three common uses of `const`:

```cpp
const double PI = 3.14159;  // this variable cannot be modified
void print(const string& s);  // 函数不能修改该变量
int size() const;  // 函数不能修改对象(对象的成员变量)
```

- `static` denotes a class-level variable shared by all objects:

```cpp
class MyClass {
public:
    static int count;  // 声明一个静态变量
    MyClass() { count++; }
};

int MyClass::count = 0; // 必须在类外定义并初始化
int main() {
    MyClass a;
    MyClass b;
    cout << MyClass::count << endl; // 输出 2
}
```
