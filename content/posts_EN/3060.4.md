# Week 6-7  Processor Architecture and Logic Design

> How do instructions actually run inside a processor?

Week 5 covered computer architecture: ISA (`x86`, `ARM`, `RISC-V`, ...).  
Week 6 turns to processor microarchitecture: different implementations under the same ISA (`single-cycle`, `multi-cycle`, `pipelined`, `out-of-order`, ...).

</br>

## **Processor Microarchitecture**

- **Data Path**: computes on data, processes it, or moves it around; it is responsible for the data flow.
- **Control**: updates and fetches instructions; it is responsible for the instruction flow.

</br>

### Cycle

- A CPU does not operate continuously. Instead, it works periodically through the cycle of “fetch, decode, execute” — fetch the instruction from memory/cache, interpret it according to the ISA, then compute or access memory. The smallest time unit is a **cycle**. Its actual duration depends on the implementation, and its frequency is measured in GHz. For example, a 3 GHz CPU processes 3×10<sup>9</sup> cycles per second.
- The CPU fetches an instruction, splits it into fields and interprets them under the ISA, executes the corresponding operation depending on the instruction type, may then access memory, write the result back to a register, and finally **increments the PC by 4**. (⚠️ Under the RV32I **C-extension**, where instructions may be 16 bits wide, the update becomes `PC += 2`.)

**Single-cycle**: **every instruction must finish within one cycle.**  
This means the cycle length must be **rounded up to the slowest instruction** — all instructions take as long as the slowest one, namely `lw`. (The fastest instructions are jumps such as `jal`.)

**Multi-cycle**: **split one instruction into multiple stages, one stage per cycle.**  
RISC helped drive the revolution toward shorter cycles, so multi-cycle execution is much more efficient in practice.

</br>

### CPU Performance Formula

⚠️ *CPU Time = Instruction Count × CPI × Clock Cycle Time*

**That is: total execution time = how many instructions must run × the average number of cycles per instruction × the duration of each cycle.**

- **Instruction count** depends on the ISA, the compiler, and the specific algorithm.
- **CPI** depends on the ISA and the processor microarchitecture.
- **Clock-cycle time** depends on the processor microarchitecture and the hardware technology.

So performance must be judged by considering all three together, not merely by counting instructions. For example, if we use an ISA with FMA, the instruction count may decrease. Under a single-cycle implementation, CPI remains 1 but the clock cycle becomes longer; under a multi-cycle implementation, CPI increases while the clock time may remain unchanged.

Overall, the contrast can be summarized as: **CISC (short, long, long) vs. RISC (long, short, short)**. Hence modern high-performance CPUs such as `x86` machines often look like this: **the ISA behaves like CISC, but internally instructions are translated into micro-ops and executed in a RISC-like manner**, combining advantages of both.

</br>

### Single-cycle, Multi-cycle, and Pipeline

**Single-cycle**: finish each instruction in one **long cycle** — CPI = 1, but the cycle itself is long.

**Multi-cycle**: let instructions proceed step by step — CPI > 1, but each cycle is shorter.

⚠️ **Pipeline**: **overlap different stages of different instructions**. For example, while the first instruction is executing, the second may be decoding and the third may already be fetching. A pipeline can be seen as the most basic form of [ILP](https://blog.nero-lithos.com/posts/3060.1/).

</br>

</br>

## Foundations of Logic Design

> In the end, a processor is just circuitry.

A low voltage near 0 V represents 0, and a high voltage near 1 V represents 1. One wire carries 1 bit; multi-bit data is represented by a bus.

</br>

### The Three Core Hardware Ingredients

**Combinational elements**: perform functional operations on the inputs and produce outputs. They are **stateless** — they do not remember.

**State / sequential elements**: store state information, such as registers and memories.

**Clock signals**: determine when the state elements update.

So the essence of “executing one instruction” is:

> Within one cycle, combinational logic computes the “next state,” and when the clock edge arrives, that next state is written into the sequential elements.

#### Basic Logic Gates

![](https://i.postimg.cc/nh30sDLz/jie-ping2026-03-03-11-03-32.png)

#### Combinational Circuit Elements

- **MUX** (Multiplexer): `Y = S ? I1 : I0`  
  If the select signal `S = 1`, output `I1`; otherwise output `I0`. It is essentially **ternary decision logic**. Any “choose one of two” or “choose one of many” problem fundamentally relies on a MUX.  
  Examples: Should the second ALU input come from a register or an immediate? Should the next PC value be `PC + 4` or the branch target?
- **ALU**: `Y = F(A, B)`  
  The control signals decide whether it performs `add`, `sub`, `and`, `or`, `compare`, and so on.
- **Comparator**: outputs `L` if `A < B`, `E` if `A == B`, and `G` if `A > B`. It is especially suitable for branch conditions.  
  ![](https://i.postimg.cc/FzfQq4nf/jie-ping2026-03-03-11-13-36.png)

A combinational circuit is always “working”: once the input changes, the output changes after some **delay**. The delay of a combinational circuit is determined by the **longest logic path from any input to any output**, which is exactly the **critical path**.

#### Sequential Elements

- **Registers**: store data, and update on the rising edge of the clock signal.
- **Register with write control**: adds write permission. If `Write = 1`, the register updates normally; if `Write = 0`, it keeps the old value. Not every instruction writes to a register — ⚠️ under RV32I, **Branch** and **Store** instructions always have `Write = 0`.

#### Clock

A valid cycle must satisfy:

> Starting from the output of one state element, the signal must pass through the combinational logic and arrive stably at the input of the next state element before the next clock edge.

The clock cannot be too fast. Otherwise the data may be written before it becomes stable, causing errors. Hence the lower bound on the clock period is determined by the **longest combinational path**.

</br>

### The “MVP Model” of an RV32I Processor

(The figure below is the smallest runnable subset of a single-cycle RV32I processor.)

⚠️ The circular branch points indicate multiple parallel wires. The **three** circled branch points from left to right <u>must be implemented with MUXes</u>:
- **PCSrc**: do we take a branch/jump?
- **ALU inSrc**: use an immediate or a register?
- **MemtoReg**: write back from memory or from the ALU?  
![](https://i.postimg.cc/vHGkNVDg/jie-ping2026-03-04-13-59-14.png)

- **R-type**, e.g. `add x3, x1, x2`:  
  `PC += 4` is computed in parallel by the left adder while the PC also supplies an address to instruction memory. The instruction is decoded, the **ALUSrc MUX** selects register input, **ALUctr** requests `x1 + x2`, <u>Data Memory is not used</u>, and the **MemtoReg MUX** selects the **ALU** result, which is written back into register `x3`.

- **I-type**, e.g. `addi x3, x1, 42`:  
  the **ALUSrc MUX** selects the **sign-extended immediate**. Everything else is similar to the R-type case.

- **Load**, e.g. `lw x3, 8(x1)`:  
  the **ALUSrc MUX** selects the sign-extended **8**, **ALUctr** computes `x1 + 8`, Data Memory reads from that address, the **MemtoReg MUX** selects **Data Memory**, and the data is written back into register `x3`.

- **Store**, e.g. `sw x3, 8(x1)`:  
  in the opposite direction, the value in `x3` is written into `data memory[x1 + 8]`, and <u>nothing is written back to the register file</u>. Otherwise it resembles the load path.

- **Branch**, e.g. `beq x1, x2, 88`:  
  the **ALUSrc MUX** selects register input, and **ALUctr uses the Comparator** to compare `x1` and `x2`. Meanwhile the machine computes both `PC + 4` and `PC + 88`, and **PCSrc** chooses between them based on the comparison result.

- **Jump**: at the **MemtoReg** side we need one extra **WBsel** (Write Back Select) element. We will discuss this later.

</br>

### Control

- A **1-bit MUX** chooses `PCSrc`
- A **1-bit MUX** chooses `MemtoReg` (is the written-back value from the ALU or from memory?)
- A **1-bit MUX** chooses `ALUSrc`
- A **1-bit Write Enable** controls memory read/write
- A **4-bit ALUctr** controls whether the ALU performs `add`, `sub`, `and`, `or`, ...  
  ![](https://i.postimg.cc/GmkmFf7P/jie-ping2026-03-04-15-57-37.png)

For example, for `lw`:
- `ALUSrc = 1` (select sign-extended immediate)
- `MemRd = 1` (need to read memory)
- `MemtoReg = 1` (write back from memory)
- `RegWr = 1` (need to write a register)

For `sw`:
- `ALUSrc = 1`
- `MemWr = 1`
- `RegWr = 0`

</br>

</br>

### Step 2: Choose Datapath Components and a Clocking Strategy

1. **CLA Adder**:

![](https://i.postimg.cc/6pWcYQfV/jie-ping2026-03-05-09-24-29.png)

2. **Register**:  
   uses the `Clk` signal as its timing source, together with write permission (`1/0`: allowed / not allowed to write)

3. **Register File**: 32 registers

- **Datapath**: two `32-bit` read ports (`busA`, `busB`) and one `32-bit` write port (`busW`)

- A 2-wide superscalar design would need at least 4 read ports; an FMA instruction would need 3 read ports.

4. **Main Memory**

- **Datapath**: one `32-bit` read port bus and one `32-bit` write port bus
- **Control**: `32-bit` address plus `Write Enable`

### Step 3: Assemble the Datapath

1. The **Instruction Fetch Unit** is always similar:
   - **fetch**: `mem[PC]`
   - **update**: `MUX: PC += 4 or PC += Imm`  
     (next instruction or branch target?)

> Is one adder enough?

— In theory some designs may reuse an adder, but in a **single-cycle** implementation one adder is generally not enough, because we often need `PC + 4` and `PC + immediate` at the same time.

> Why can’t `beq` simply use something like `sub rs3, rs1, rs2 = 0` instead of a comparator?

— **Overflow**. If subtraction is used for comparison, we must introduce additional flags such as **V (overflow)** or **C (carry)** to disambiguate the result.

2. The remaining execution modules should be integrated into a minimal multiplexed circuit using MUXes:

![](https://i.postimg.cc/vHMp9fDS/jie-ping2026-03-05-12-08-52.png)

> The core principle of MUXes: **do not imagine building a separate circuit for every instruction. Reuse the datapath as much as possible, and configure it through control signals.**

For example, the figure above typically merges the datapaths of memory instructions and R-type instructions.

3. **Context switching is essentially “switching state.”**

> **We do not save every single thing inside the machine. We save and restore only the architectural state necessary to resume this execution stream** — including the PC, Register Files (general-purpose and special registers), `SP` (stack pointer), and `RA` (return address).

- **Thread switch** saves processor registers
- **Process switch** saves the PCB (including registers)

> Microarchitectural state such as cache contents, control signals, or pipeline state does **not necessarily** need to be saved.

4. The whole circuit:  
   (adding the path needed by `jal`: jump to the target address and write the return address `PC + 4` into `rd`)  
   ![](https://i.postimg.cc/XYdqxX58/jie-ping2026-03-05-15-08-53.png)

5. Clock period:  
   <u>A register can be read during the same cycle, pass through combinational logic, and then be written back at the end of that same cycle.</u>

> The read happens during signal propagation after the clock edge; the write happens before the next clock edge.

- “The lower bound on the clock period is determined by the **longest combinational path**.”  
  For a single-cycle CPU, the slowest path is typically the one for `lw`, whose propagation delay includes:

> Critical Path = "PC's Clk-to-Q" + "Instruction Memory Access Time" + "Register File Access Time" + “ALU 32-bit Addition Time” + “Data Memory Access Time" + "Register File Setup Time" + "Clock Skew".

(`PC's Clk-to-Q`: after the clock arrives, the PC output takes some time to stabilize.  
`Setup time`: before writing into the register file, the input must remain stable for a short period.  
`Clock skew`: the clock signal arrives at different components at slightly different times.)

- Hence the frequency limit of a single-cycle CPU is usually determined by the slowest path, namely `lw`.

</br>

### Obvious Weaknesses of the Single-Cycle Processor

- Since there is no ILP and no resource reuse, multiple redundant adders are required (`PC` update and `ALU` arithmetic need separate hardware).
- The same combinational or state element cannot be traversed multiple times within one cycle.
- The register file does not need to be duplicated, but it does need multiple ports (`2 read, 1 write`).
- `mul` and `div` are iterative computations; to force them into a single cycle, their internal repetition would have to be fully unrolled, which is extremely expensive.

Therefore, the more realistic solution is to allow **multi-cycle execution**, where a counter controls repeated reuse of the same hardware resources. The trade-off is that the CPI of those instructions increases.

</br>
