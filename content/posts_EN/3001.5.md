# LN6 GCD: Greatest Common Divisor

<br>

## Basic Concepts

<br>

- Greatest common divisor: if an integer c divides both a and b, i.e. c | a and c | b, then c is a common divisor of a and b. The greatest such divisor is called the **gcd**, written gcd(a, b). If a and b have no common divisor other than 1, then gcd(a, b) = 1, and they are called relatively prime.
- Quotient-Remainder Theorem: for any integer a and positive integer b, there exist unique integers q and r such that a = q·b + r and 0 ≤ r < b. We usually write q = a div b and r = a mod b.

<br>

## Euclid’s Algorithm

<br>

- If a ≥ b > 0:  
  if a = kb, then gcd(a, b) = b immediately. Otherwise write a = qb + r with r > 0. The key idea is:
  gcd(a, b) = gcd(b, r) = … = gcd(g, 0)
  Continue recursively until the remainder becomes 0; the current nonzero value g is the gcd of a and b.
- Example: $$gcd(728, 476) = gcd(476, 252) = gcd(252, 224) = gcd(224, 28), gcd(28, 0) = 28$$

<br>

## Bézout's Identity

<br>

- Define an **integer linear combination** as an expression of the form "sa + tb" where s, t ∈ Z.  
  **Theorem (Bézout’s Identity):**
  gcd(a, b)= min{ |sa + tb| > 0}  
  that is, the gcd is the smallest positive integer linear combination, where s or t may be negative.  
  Write this quantity as spc(a, b). The proof has two parts:
  $$(i)\ \gcd(a,b)\mid \mathrm{spc}(a,b)\Rightarrow \gcd\le \mathrm{spc};\ \ (ii)\ \mathrm{spc}(a,b)\text{ also divides }a,b\Rightarrow \mathrm{spc}\le \gcd. $$
  Hence the two are equal.

- **Corollary of Bézout’s Identity**: if gcd(a, b) = 1 and gcd(a, c) = 1, then gcd(a, bc) = 1. This follows by multiplying the two Bézout equations and rearranging.

- Another corollary: if gcd(b, c) = 1 and c | ba, then c | a.

  Since gcd(b, c) = 1, there exist s and t such that sb + tc = 1, so sba + tca = a. Because c | ba, we have c | sba; and clearly c | tca. Therefore c | a.

<br>

## Extended Euclid

<br>

- After computing the gcd, we can work backward through the Euclidean algorithm to recover the corresponding Bézout coefficients.
- Example: still using 728 and 476, trace back from gcd = 28 to obtain spc(728, 476):
  $$28=252-224=252-(476-252)=2\cdot252-476=2(728-476)-476=2\cdot728-3\cdot476$$

<br>

## Prime Divisibility (PD) and the Fundamental Theorem of Arithmetic (FTA)

<br>

- If p is prime and p | ab, then p | a or p | b. Using Bézout:
  $$If\ p
mid a\ 则\ \gcd(p,a)=1\Rightarrow\exists s,t: sp+ta=1，乘以\ b\ 得\ sbp+tab=b，$$$$因为\ sbp\ 含有\ p,\ p\ |\ sbp；又因为\ p\ |\ (ab\ |\ tab),于是 p\mid b$$
- FTA (uniqueness of factorization): every n > 1 can be written as a product of prime factors in increasing order, and this factorization is unique.

<br>

## The “Die Hard” Water Jug Problem (Invariant Method & Linear Combinations)

<br>

With two jugs of capacities a and b, **after every operation, the amount of water in the jugs remains an integer linear combination of** a **and** b. Therefore every reachable amount must be a multiple of gcd(a, b). So 4 liters cannot be measured using jugs of capacities 3 and 9 (because gcd = 3), but 4 liters can be measured using capacities 21 and 26 (because gcd = 1). **A target volume** k **is measurable if and only if it is a multiple of** gcd(a, b).

# LN7 Modular Arithmetic

<br>

## Congruence

<br>

- The mod operation is just taking a remainder, for example 7 mod 2 = 1.
- A useful simplification identity:
  an mod bn ≡ n(a mod b)  (a, b, n ∈ Z)

- Definition of congruence:
  a ≡ b (mod n) ⇔ n | (a-b). This is equivalent to saying that **a and b have the same remainder when divided by** n.
- Relationship between "mod" (remainder) and “≡” (congruence):
  a ≡ b (mod n) ⇔ a mod n = b mod n

<br>

## Modular Addition and Modular Multiplication

<br>

- If a ≡ c and b ≡ d (mod n), then a+b ≡ c+d (mod n) and ab ≡ cd (mod n).

    Proof idea: substitute a=c+nx and b=d+ny, then rearrange to show that n | (a+b-c-d) or n | (ab-cd).

- **Modular exponentiation**: by closure under multiplication, a<sup>m</sup> ≡ c<sup>m</sup> (mod n). More generally, for any polynomial f with integer coefficients, f(a) ≡ f(c) (mod n).

- **Computational technique in modular arithmetic**: repeatedly reduce modulo n to simplify large expressions, especially when handling large powers.

<br>

## Applications of Modular Arithmetic: Divisibility Tests

<br>

- **Divisibility by 9 or 3**: since 10<sup>m</sup> ≡ 1 (mod 9) and 10<sup>m</sup> ≡ 1 (mod 3) for m ∈ Z<sup>+</sup>, a number is congruent to the sum of its decimal digits modulo 3 or 9.
  For example,
  252 = 2x10<sup>2</sup>+5x10+2x1 ≡ 2+5+2 ≡ 9, so 9 | 252.

- **Divisibility by 11**: since 10<sup>m</sup> ≡ (-1)<sup>m</sup> (mod 11), the decimal expansion becomes an alternating sum of digits.
  For example,
  121 ≡ -1+2+-1 = 0, so 11 | 121.

<br>

## Multiplicative Inverse

<br>

- Definition: modulo n, the **inverse** of a, written a', satisfies aa' ≡ 1 (mod n). In this discussion, all elements and inverses are taken modulo n.
  For example, the inverse of 3 modulo 10 is 7, since 21 ≡ 1 (mod 10).

<br>

- **Necessary and sufficient condition for existence of an inverse**:
  gcd(a, n) = 1 ⇔ an inverse exists.
  The proof uses Bézout: if gcd(a, n) = sa+tn = 1, then s is an inverse of a modulo n.
  For example, 2 has no inverse modulo 10 because gcd(2, 10) = 2.

<br>

- Z<sub>n</sub> denotes the set of residues modulo n together with their multiplicative behavior. For example, in Z<sub>6</sub>, 2, 3, and 4 have no inverses. Under a prime modulus, every nonzero element has an inverse.

<br>

- **Cancellation**: in modular arithmetic, we generally **cannot** directly cancel a common factor.
  For example, 3x2 ≡ 5x2 (mod 4), but 3 and 5 are not congruent modulo 4.
  Cancellation is valid **if and only if** gcd(common factor, modulus) = 1, because in that case the common factor has an inverse, and we can multiply both sides by it. For example:
  6x3 ≡ 11x3 (mod 5) → 6x3x3' ≡ 11x3x3' (mod 5) → 6 ≡ 11 (mod 5)

<br>

## Fermat’s Little Theorem & Wilson’s Theorem

<br>

- **Fermat’s Little Theorem**: if p is prime and gcd(k,p)=1, then
  **k<sup>p-1</sup> ≡ 1 (mod p)**

  Proof idea: the set {k,2k,…,(p-1)k} modulo p is just a permutation of {1,…,p-1}. Multiplying all terms together and cancelling modulo p yields the result.

- A useful consequence:
  **k<sup>x</sup> ≡ k<sup>x mod (p-1)</sup> (mod p)**.
  For example, 2<sup>81</sup> ≡ 2 (mod 11).

  Proof idea: multiplying by any number of factors of k<sup>p-1</sup> does not change the remainder, so such factors can be removed.

<br>

- **Wilson’s Theorem**:
  **p is prime ⇔ (p-1)! ≡ -1 (mod p)**.
  Necessity: if p is not prime, then Z<sub>p</sub> contains noninvertible divisors, so (p-1)! ≡ 0 (mod p).
  Sufficiency: if p is prime, then every element other than 1 and p-1 can be paired with its inverse, and each pair multiplies to 1, leaving 1 x (p-1) ≡ -1.

<br>

## Linear Congruences of the Form ax ≡ b (mod n)

<br>

- **Necessary and sufficient condition for solvability**:
  gcd(a,n) | b.
  If gcd(a,n)=1, multiply both sides by a<sup>-1</sup> to get the solution.
  If gcd(a,n)=c>1, then a solution exists if and only if c | b, in which case the equation can be reduced to a solvable congruence modulo n/c.
  The solution set forms an arithmetic congruence class.
