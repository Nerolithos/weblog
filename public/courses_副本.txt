- **CSC1001 Programming Methodology (Introduction to Computer Science: Programming Methodology)**
  - 老师：刘桂良 (Guiliang Liu)。
  - 课程描述/评价：这门课用 Python 讲授编程基础，包括基本语法、数据结构、算法设计等。课程设置四次编程作业、一次期中测验和一次期末考试，分数分别占 40%、20% 和 40%。作业要求严格，每次延期提交将被扣除总分的 10%；助教不鼓励随意退课，需经教师和学院批准。课程鼓励同学讨论，但要求独立完成作业，抄袭会受到严厉处罚。教师明确说明只有成绩达到 A 或 A‑且积极参与课程的同学才会获得推荐信。总体评价是工作量大、考试难度中等，只要诚实完成作业便能获得不错的成绩。

  

- **CSC1002 Computational Laboratory**
  - 老师：由数据科学学院安排，网页未列出具体教师。
  - 课程描述/评价：这是一个 1 学分的编程实验课，使用 Python 和相关工具，通过一系列实验提升编程能力。作业难度逐步提升，需要学生独立完成。课堂主要用于实操，不常点名，但实验报告必须按时提交，给分一般较友好。

  

- **CSC1004 Computational Laboratory (Java 与 Python)**
  - 老师：刘桂良 (Guiliang Liu)。
  - 课程描述/评价：课程旨在提高编程能力，需要学生独立完成 2 个中等规模的 Java 项目（每个约 2000 行代码）和 1 个较小的 Python 项目（约 500 行）。评分由 Java 项目 40%、Python 项目 20% 和期末考试 40% 组成。作业每晚迟交一天扣 10%。教师不鼓励中途退课，必须经教师和学院批准。抄袭会受到严厉处理，期末考试禁止使用 AI 工具或第三方库。课程工作量大，编程项目要求高，但认真完成可以获得较高分数。

  

- **CSC3002 Programming Paradigms (C/C++ 编程范式)**
  - 老师：黄锐。
  - 课程描述/评价：课程从 Python 过渡到 C++，涉及递归、动态内存管理、图结构、堆、二叉树等高级数据结构和算法。课程比较不同编程范式，对 C/C++ 与其他语言的优缺点进行分析。教师对作业要求严格，迟交会扣分；对“晚退课申请”很严格，建议学生在选择课程时充分考虑工作量。课程难度较大，需要投入大量时间完成编程练习，但评分相对公平。



- **CSC3050 Computer Architecture (计算机体系结构)**
  - 老师：许维中 (Hsu Wei Chung)。
  - 课程描述/评价：课程介绍指令集结构、流水线、存储层次等计算机体系结构知识，并包含 Verilog/硬件设计实践。课程不会强制点名，但建议参与，否则可能损失参与分。教师通过 Piazza 平台答疑，强调提问前应阅读资料和自行搜索，培养独立解决问题的能力。课程网页《如何在 CSC3050 取得成功》指出该课不容易拿到 A，大部分学生以掌握知识为目标；建议学生尽早开始项目、自主调试、阅读教材，强调很多学生不会得 A，重在学习。总体评价为工作量大、学习曲线陡峭，需要自律和大量实践。

- 关于学生证：期末考试硬性规则需要出示学生证，如若丢失，需要当场填一式两表，保留其下联交给 TA1 的教务处。但是，令人费解的是，你无法通过身份证、护照、驾驶证、电子学生证中的任何一者或多者作为提交时的身份证明。你需要在 48 小时内前往中信银行在下沉广场的网点补办学生证（30 元）。注意，当场交给你的学生证不会立即生效。如果超出 48 小时，本科成绩记作零分。这是个巨大的行政漏洞，在可见的未来不会改正。

- 抢课与 AD 周：学校在 sis 系统上的抢课系统存在一个显然的漏洞，即，虽然教务处规定不参与预选课的学生将延迟 10 分钟开始抢课作为惩罚，但是并没有规定要按照自己的真实需求预选课，所以很多人会在预选课时只报一两门充数，然后在正式选课时加上大量未预选的科目。这就导致了教务处在使用预选课统计每门课的意向报名人数时数据的巨大偏差。解决这个问题的方案比如将学院必修课（核心课）改为直接安排给学生，只开通选修课（CEC、ENG、CHI 等）的抢课。

  AD 周一般有两个阶段，在学期开始当日或者两天之内的某日，即 AD 周的第一天，教务处会开放剩余名额，学生可以以普通选课方式加课。在 AD 周中间某日起开始线上交表，建议与意向课程教授事先沟通，因为教授对交表的通过与否有最大话语权。交表流程 Registry 会用邮件通知，成功的话也会另行邮件告知。
