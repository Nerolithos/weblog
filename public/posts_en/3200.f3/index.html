  • The ampersand in declarations: char &r = o; means a reference bound to an existing variable o. A reference is effectively another name for the same object, so changing either one changes the same underlying value. char *&p means a reference to a pointer, whereas char &*p is illegal.
  • As operators, * and & mean dereference and address-of, respectively, so *&v == v and &*p == p.
  • Increment and decrement operators: ++v increments before returning; v++ returns first and then increments. As simple loop iterators, the distinction often does not matter. But expressions such as *++p, ++*p, *p++, and (*p)++ have importantly different meanings.
  • Memory: in a function, char arr1[42]; allocates directly on the stack, while auto arr2 = new char[42]; allocates on the heap.   • Arrays: an array object such as char arr[42]; implicitly decays into a pointer to its first element, so *arr and arr + n are legal. But arr++ is illegal: an array is not literally the same thing as a pointer. By contrast, after auto ptr = arr;, ptr[n] is legal.

  1. Time Complexity

  • Three basic sorting methods：

    Selection Sort repeatedly picks the nth smallest remaining element and swaps it into the nth position. For x elements, it needs x-1 comparisons to identify the minimum, so the total is $(n-1)+(n-2)+…+1+0={n(n-1)\over 2}$ comparisons, which stays at O(n2). Independent of initial ordering.

    Insertion Sort scans from left to right and shifts each element leftward until it reaches a smaller predecessor. As above, it may require up to ${n(n-1)\over 2}$ moves, so the average / worst case is O(n2), but if the data is already sorted it needs only one pass, giving the best case O(n). The more ordered the input is, the faster it becomes.

    Bubble Sort repeatedly swaps adjacent out-of-order pairs from left to right, usually across multiple rounds. It may perform up to ${n(n-1)\over 2}$ swaps, so the average / worst case is O(n2), while an already sorted input can be handled in O(n) if an early-stop check is added. The more ordered the input is, the faster it becomes.

  • Four divide-and-conquer / logarithmic-structure sorting ideas：

    Quick Sort places smaller elements on one side of a pivot and larger elements on the other, then recurses on both parts. Its average / best complexity is O(nlogn), but if the pivot repeatedly becomes the maximum or minimum, it degenerates toward a selection-sort-like worst case.ction-sort-like worst case**, namely O(n2).ivot 有关**

    BST Sort inserts the elements into a binary search tree and then outputs them by inorder traversal. As above, the best / average case is O(nlogn), but if the input sequence is already ordered the tree may degenerate into a linked list, yielding the worst case O(n2). The more ordered the input is, the worse it may become.

    Merge Sort recursively splits the data left and right until only one element remains, then merges upward in sorted order. Because it does not depend on pivot choice or BST structure, it does not degenerate and stays at O(nlogn). Independent of initial ordering.

    Heap Sort builds a min-heap and then performs n calls to deleteMin; building the heap costs O(n), and each deletion costs O(logn), so the total remains O(nlogn). Independent of initial ordering.

  • In average practice: Quick Sort ≲ Merge Sort ≲ Heap Sort ≲ BST Sort ≪ Insertion Sort ≲ Bubble Sort ≲ Selection Sort.



  1. OOP

  • The three major pillars of OOP are Encapsulation (packaging methods and data into a class), Polymorphism (compile-time overloading and run-time overriding), and Inheritance (subclasses inheriting members from parent classes).

  • Overloading: parameter types and parameter counts are part of the signature, but ⚠️ the return type is not.

  • Inheritance: in class Son : public Father, the subclass inherits members according to the chosen access mode; inheritance mode affects how inherited members are exposed.

    Virtual inheritance can resolve the ambiguity of the diamond inheritance pattern by ensuring that only one shared virtual base subobject is created.

    Virtual functions enable run-time polymorphism: when a base-class function is declared virtual, calling it through a base-class interface can dispatch to the subclass implementation.

  • Operator overloading must follow the language rules; one cannot arbitrarily define every possible form. Standard types such as std::string, as well as user-defined struct, enum, or class types, may support appropriate operator overloading.

  • For class Example {int a; char* b;};, the constructor initializer form can be written concisely as public: Example (int x, char* y) : a(x), b(y) {}.

  • 用平衡条件让二叉搜索树不超过 logn 级别的高度，就不会退化。

  1. Heap

  • 堆是 “按优先级处理元素” 的数据结构，所以又叫 Priority Queue。堆不同于 BST，它不把所有元素整体排序，但堆可以用 BST 实现。分为 MinHeap 和 MaxHeap，优先处理最小或最大项。依旧需要 left, right, parent (索引)，接口：insert()、push()、deleteMin() / deleteMax()。

  • 建立在 vector 基础上的二叉堆 Binary Heap 是最优解，插/删 O(log N)，findMin O(1)。二叉堆必须表现为一棵完全二叉树，并用动态数组形式存储 (⚠️不需要指针！)，二叉堆有两条性质：

    结构性：对于任意索引 i (⚠️存储在 vector 中，所以索引从 0 开始)，其父子节点可以直接计算：parent(i) = (i−1)/2；left(i) = 2i+1；right(i) = 2i+2，这样把二叉堆画成 二叉树后索引从左到右从上到下逐行增大，且为0～N 的连续整数表示。

    有序性 (heap-order property)：对 MinHeap，所有父节点小于子节点，MaxHeap 反之。堆只保证 “父子有序”，左右是无序的，故中序遍历不会得到升/降序列。

    ⚠️一棵高 h 的完全二叉树：N∈[2h, 2h+1-1]

    插入 v：先加入动态数组尾部 minH.push_back(v) 然后上滤 (siftUp)： std::swap(minHeap[idx], minHeap[parent(idx)]); idx = parent(idx); 也就是向 parent 方向反复交换直到 parent 小于自己。

    删除根 (deleteMin)：先用数组尾项代替根 minH[0] = minHeap.pop_back(); 然后下滤 (siftDown)：循环交换自己与左右孩子中较小的那个，直到左右孩子都比自己大。⚠️Note:每次交换先要确定左右孩子存在 left(idx) < size() 防止下标越界。

    buildHeap 建堆：先将元素写入动态数组 for (int i = 0; i < vec.size(); i++) {data_.push_back(vec[i]);}，然后从最后一个非叶节点开始下滤 for (int i = size() / 2 - 1; i >= 0; --i) {siftDown(i)} (数组末尾在 N-1，其父节点就在 N/2-1)。总共大约在O(N)。



  1. Graph

  • 树是没有循环的图，图是一种满足度数和为偶数的 G~(E, V) 结构。

    无向图有边的两点互成邻接，有向图 (digraph) 被指向的点邻接指向它的点。

    实现图可以用 adjacency list：每个顶点做一个数组/链表，再把顶点做成桶数组 (Map)，遍历复杂度 O(|V|+|E|)；也可以用 adjacnecy matrix：用 |V|x|V| 的二维数组 [i][j] 表示 i 到 j 有边，无向图是对角对称的，遍历复杂度 O(|V|2)。

    DFS：用 stack/递归 先走一条路径到底，然后回溯访问其他路径。思路类似于没有左右的 preorder

    BFS：用 queue 依次走距离近到远的点。

    最短路径问题：无权时用 BFS 遍历记下到根的层数；有权需要用 Dijkstra 算法：从 r 开始，对当前点所有相邻点，如果当前点到 r 的距离+边权重比记录值更小，就更新距离，如此多轮。

    Tropological Sorting： 对 DAG (有权无环图)，如果 (u, v) 有边，先访问 u (先访问所有前序点)。
