LN5 Recursion

Recursion is the underlying logic of mathematical induction, and it is also a common technique in coding.


Recursive Thinking

Cardinality of the Power Set of a Set

For Sn = {a1, a2,…, an}, what recurrence relation does |pow(Sn)| satisfy? S1 = {ø, {a1}}, S2 = {ø, {a1}, {a2}, {a1, a2}} …… So Sn+1 contains all elements already in Sn, together with the unions of those elements with an+1. (Here Se denotes the individual elements of S.) $$S_{n+1}={{Se_n, Se_n∪{a_{n+1}}}}$$ Therefore |pow(Sn+1)| = 2|pow(Sn)|


Binary Strings Without “111”

What recurrence counts the number of n-bit binary strings Bn that do not contain the substring “111”? B1：{1, 0} →2 B2：{10, 01, 11, 00} →4 B3：{110, 000, 010, 011, 001, 100, 101} →7 B4：{1100, 0011, 1011, 1101, 0000, 1000, 0001, 0101, 1010, 0100, 0110, 0010, 1001} →13 …… Observe the elements of B4:

  • {0110, 0000, 0010, 0011, 0001, 0100, 0101} come from B3, and all of them can be obtained by adding a leading 0. {1010, 1001, 1011, 1000} come from B2, and all of them can be obtained by adding the leading bits 10. {1101, 1100} come from B1, and all of them can be obtained by adding the leading bits 110.
  • Equivalently, one may say: “every element of B3 may have a leading 0 added, while those whose first two bits are not 11 may also have a leading 1 added.” Hence |Bn| = |Bn-1| + |Bn-2| + |Bn-3| for n>3.


Solving Problems Recursively

Core idea: reduce the large to the small, unify the many into one, and divide the whole into parts.


Towers of Hanoi

There are three pegs, and n disks stacked on the leftmost peg from largest to smallest. Under the rule that on every peg a larger disk must always remain below a smaller one, how many moves f(n) are needed to reproduce the entire stack on the rightmost peg without changing the order?

  • For brevity, let the smallest disk be 1 and the largest be n; name the pegs a, b, and c.

  • n=1: 1->c

  • n=2: 1->b、2->c、1->c

  • n=3: 1->c、2->b、1->b、3->c、1->a、2->c、1->c

Looking at n=3, the idea is simple: first move the two smaller disks away, then move the largest disk to the rightmost peg, and finally move the two smaller disks onto it. The middle step is equivalent to n=1, while the first and last parts both amount to moving two disks, in the same relative order, from one peg to another (“from a to b” and “from b to c”). Since the three pegs have no intrinsic individuality, the number of steps depends only on the number of disks, not on which peg is the source or the destination. Thus, analyzing f(3) amounts to calling f(2), then f(1), then f(2). So we may conjecture that f(3) = 2f(2)+1, and in general f(n) = 2f(n-1)+1, as illustrated below.

Following the same logic, every Towers of Hanoi problem can be solved recursively by a three-part decomposition, until all that remains are individual moves. Therefore, in Python we essentially only need a single-move routine, a three-way recursive routine, an entry routine, and the main routine:

def move(src: list[int], tar: list[int], name_src: str, name_tar: str, steps: list[int], A, B, C):
    # 移动一个圆盘，并打印当前状态
    pan = src.pop() # 记录并去掉 source 数组顶部的元素
    tar.append(pan) # 将其加到 target 数组
    steps[0] += 1
    print(f"第 {steps[0]} 步: {name_src} -> {name_tar}")
    print(f"A={A}, B={B}, C={C}
")

def dfs(i: int, src: list[int], buf: list[int], tar: list[int], # 递归 f(i)
        name_src: str, name_buf: str, name_tar: str,
        steps: list[int], A, B, C):
    if i == 1:
        move(src, tar, name_src, name_tar, steps, A, B, C)
        return
    dfs(i - 1, src, tar, buf, name_src, name_tar, name_buf, steps, A, B, C)  # f(i-1)
    move(src, tar, name_src, name_tar, steps, A, B, C)  # f(1)
    dfs(i - 1, buf, src, tar, name_buf, name_src, name_tar, steps, A, B, C)  # f(i-1)

def solve_hanota(A: list[int], B: list[int], C: list[int]):
    # 求解并返回总步数
    n = len(A)
    steps = [0]
    dfs(n, A, B, C, "A", "B", "C", steps, A, B, C)
    return steps[0]

if __name__ == "__main__":
    num = int(input("请输入汉诺塔圆盘数量："))  # 注意 input 自动存储为 type:str
    A = list(range(num, 0, -1))  # [num, num-1, ..., 1]
    B = []  # 也可以写 list[int]
    C = []
    total_steps = solve_hanota(A, B, C)
    print(f"共移动了 {total_steps} 步。")

Catalan Numbers

Let the recurrence be denoted by m(n). After some observation, one finds that the “peak” of every mountain has the shape “/\". We may split a mountain into two components: “U”, the mountain body beneath the peak, and “E”, the peak itself (the pattern “/\”). For instance, when n=3, the possibilities can be abstracted as EEE, EUE, UEE, UEE, and UUE. A convenient way to think about this is as a binary decomposition: one part is placed “on top of U”, while the rest is distributed freely.

  1. “E …remaining n-1 parts…” : m(0) x m(n-1)

  2. “Uone part…remaining n-2 parts…” : m(1) x m(n-2)

  3. “Utwo parts…remaining n-3 parts…” : m(2) x m(n-3)

k. “Uk-1 parts…remaining n-k parts…” : m(k-1) x m(n-k)

$$Adding\ all\ cases\ together\ gives\ the\ Catalan\ recurrence:\ m(n) = \sum_{k=1}^{k=n} m(k-1) imes m(n-k)$$

$$It\ is\ also\ written\ as:\ Catalan_n = rac1{n+1}inom{2n}{n} = rac{(2n)!}{(n+1)!,n!}$$



Solving Recurrence Relations

Closed forms for first-order linear recurrences

  • Homogeneous linear recurrence (coefficient of the previous term ≠ 1): $$a_k=r a_{k-1}\ o\ a_k=r^{k-1}a_1$$
  • Nonhomogeneous baseline linear recurrence (coefficient of the previous term = 1): $$a_k = a_{k-1} + b\ o\ a_k = a_1 + (k-1)b$$
  • Nonhomogeneous linear recurrence with coefficient ≠ 1, such as Hanoi: $$a_k=r a_{k-1}+b\ o \ a_k=r^{k-1}a_1+ rac{b,(r^{,k-1}-1)}{r-1}$$

Nonlinear single-variable recurrences of divide-and-conquer type

For a divide-and-conquer recurrence of the form a2k = mak+nk+c, where C is a constant determined from the initial condition:

  • General solution (m eq 1,2): $$a_{2k} = C·2k^{log_2m} + rac{n}{2-m},2k + rac{c}{1-m}$$
  • Special case m=2: $$a_{2k} = C·2k + nklog_22k - c$$
  • Special case m=1: $$a_{2k} = a_1 + n(k-1) + ct$$

For example, for merge sort, a2k = 2ak+2k = 2klog2(2k).


Generating-function method

For a first-order recurrence defining a sequence a, with a general form such as $$a_{n} = c_{n-1}·a_{n-1} + c_{n-2}·a_{n-2},\ n≥2$$

one may assume the generating function is $$F(x) = \sum_{n=2}^{\infty} a_nx^n$$

Then the recurrence can be written in generating-function form as $$F(x)-a_0-a_1x=c_{n-1}x·(F(X)-a_0)+c_{n-2}x^2·F(x)$$

Hence one obtains $$F(x)={{a_0+a_1x-a_0c_{n-1}x}\over{1-c_{n-1}x-c_{n-2}x^2}}$$

Factor the quadratic denominator, decompose it into the sum of two fractions, and determine their numerators by matching coefficients with the original numerator.

Then recover the generating function itself using the identity $$ raceta{1-lpha x}=\sum_{n=0}^{\infty}eta·lpha^n x^n$$

Finally, once the generating function has been expanded, one can read off the closed form of the sequence by removing the summation symbol and the factor xn.
