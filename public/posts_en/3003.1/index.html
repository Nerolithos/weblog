D3 Fundamentals

D3 (Data-Driven Documents) is a JavaScript library used to map data onto web pages and create interaction/animation.

The standard building blocks of a web page are:

  1. HyperText Markup Language (HTML) — the rendering language of the web page
  2. Document Object Model (DOM) — the manipulable page structure written in HTML
  3. Cascading Style Sheets (CSS) — visual presentation (color, font, size, layout, etc.)
  4. Scalable Vector Graphics (SVG) — the vector graphics system
  5. JavaScript-D3 — the core layer for storing/processing data and making the page dynamic


HTML and CSS

The DOM is the skeleton of a web page written in HTML, and it has a tree-like structure. In HTML, paired container tags such as <...> ... </...> indicate that “the following content belongs to component xxx; component xxx ends here.” Its basic structure has three parts: declaration, head, and body.

<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Page Title</title>
  </head>
  <body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
    <script src="app.js"></script>
  </body>
</html>

Common Components of <head>

  • <title> — the title shown on the browser tab
  • <meta> — metadata that tells the browser some non-visible page configuration information, such as the character set or responsive viewport settings.

CSS

This determines the visual appearance of components inside the body.

  • external CSS: use <link rel="stylesheet" href="styles.css" />, then edit the rules in the external file styles.css.
  • inline CSS, for example:
<style> 
h1 {color: blue; font-family: verdana; font-size: 300%; } 
p {color: red; font-family: courier; font-size: 160%; }
</style>

Common Components of <body>

  • <header>: page header (logo, navigation)
  • <nav>: navigation bar
  • <main>: main content (usually only one main per page)
  • <section>: a content section or chapter
  • <article>: an independent article or post
  • <aside>: sidebar or supplementary information
  • <footer>: page footer (copyright, links)
  • <h1>~<h6>: headings of different levels
  • <p>: a paragraph of text
  • <br>: line break
  • <div>: a general-purpose container (division) used for layout and grouping
  • <script>: the JavaScript portion of the page (this is where D3 lives) A common layout for an HTML visualization project is:
<body>
  <h1>DDA3003 Project</h1>
  <div id="controls">
    <!-- dropdowns, buttons, sliders -->
  </div>
  <div id="chart"></div>  <!-- D3 injects SVG or Canvas here -->
  <script src="https://d3js.org/d3.v7.min.js"></script>
  <script src="main.js"></script>
</body>


Javascript Fundamentals

JavaScript is a highly readable language with strong OOP flavor; compared with the C family, it is somewhat closer to Python. In the <script> part of <body>, JS (together with D3) is used to organize data.

  • Declare a global variable with x = 42, a constant with const x = 42, and a local variable with let x = 42 (or var x = 42).
  • Semicolons are optional.
  • Single and double quotes are both acceptable.
  • As in Python, you do not need to declare variable types explicitly.
  • Output function: console.log()
  • [x, y, z] represents an array, and {a: b, c: d} represents a dictionary-like object, very similar to Python.
let arr = [1, 2, 3];         // array
let person = { name: "A" };  // object
let f = function() {};       // Function(also an object)
  • Objects are described in key–value form; methods are no exception:
function cat(name, sex, type) {
	this.name = name;
	this.sex = sex;
	this.type = type;
}
const myCat = new cat('Gogo', 'female', 'Ragdoll');
console.log(myCat.type);
  • Arrays, functions, and similar objects are often declared with const. An array is a special kind of object; it can contain arrays or dictionaries as elements, and indexing starts from 0:
const cats = ['Ragdoll', 'Simease', 'BritishShortHail'];
cats[0] = 'FoldedEar';
cats.push('FoldedEar');  // append to the end of the array
cats.pop();   // return the last element and remove it
console.log(cats.slice(1, 3))  // shallow copy of elements at indices 1 and 2 (`slice[a, b)`)
cats.shift();  // remove the first element
cats.unshift('Ragdoll');  // add to the front of the array
const mapMeow = cats.map(x => x + ': Meow!');  // map over the whole array
  • Comparison: forEach is for doing things (side effects), whereas map creates a new array (pure transform).
data = [42, 4, 2, 4242];
const newData = data.map(function(x) {return x + 42;});
data.forEach(x => console.log(x));
const mean = data.reduce((sum, x) => sum + x, 0) / data.length;  // mean


SVG Fundamentals

SVG provides many basic shape-drawing primitives:

<svg width = "500" height = "500">
  <circle cx = "40" cy = "20" r = "30" stroke = "yellow" stroke-width = "2" fill = "white">
  </circle>
  <rect>   <!-- similarly: x, y, width, height, ...-->
  <line>   <!-- similarly: x1, x2, y1, y2, ...-->
  <path>
  <text>   <!-- similarly: x, y, text-anchor, ...-->
  <g>      <!-- group; create one with `group = vis.append("g")`, then add elements via `group.append()` -->
</svg>
