August 31, 2025

(Covers Lecture Note 1)

Discrete Mathematics


  • What is it?

    A branch of mathematics that studies countable, discontinuous, and independent objects.

  • What does it study?

    Propositions, logic, mathematical proofs, set theory, functions and mappings, combinatorics, graph theory, groups, complexity (the P/NP problem), and more.

  • Why do we study it?

    The underlying logic of computer systems and the underlying logic of discrete mathematics are both built on Boolean operations, Boolean functions, and ultimately 0 and 1.



LN1 Propositions and Operations


Induction: Propositional Logic

  1. Base Case Prove that the proposition holds for the smallest natural number, such as n=1 or n=0.
  2. Inductive Hypothesis Assume that the proposition holds when n=k.
  3. Inductive Step Under that assumption, prove that the proposition also holds when n=k+1.


Boolean Algebra

Boolean algebra is analogous to ordinary algebra, except that it has only two discrete values: true (1) and false (0).

Boolean operators:

  • AND (∧) corresponds to the product (×), because truth is rarer and the resulting term is called a minterm: since any 0 makes the result 0, just as in arithmetic only 1×1=1.
  • OR (∨) corresponds to sum (+) without overflow, because truth is more permissive and the resulting term is called a maxterm: as long as one input is 1, the result is 1, just as 1+0=1 and 1+1=1 in Boolean arithmetic.
  • NOT (¬) corresponds to complementation, similar to 1−x.
  • XOR (⊕) corresponds to sum (+) with overflow: 1+0=1, while 1+1=2=0.

Functional completeness means that a collection of operators is sufficient to express all logical operators. For example, {AND, NOT} is not complete, but {AND, NOT, OR} is complete.



Logic Operators

  • A proposition or statement is a declarative sentence that is definitely true or definitely false, without requiring additional conditions.

  • x>0 is not a valid proposition.

  • An operator is a Boolean function describing a logical operation in Boolean algebra.

  • A truth table lists every possible combination of truth values together with the output of a given logical function. The three basic operators are: $$非：NOT\ P=¬P= \overline{P}$$ $$与：P\ AND\ Q=P∧Q\ \ (iff\ both\ true\ then\ true)$$ $$或：P\ OR\ Q=P∨Q\ \ (iff\ both\ false\ then\ false)$$

  • Every operator can be expressed using these three basic operators. There are two canonical forms for constructing logically equivalent formulas for all operators:

    1. Apply Sum-of-Products (SOP, AND first, then OR) only to the rows where the output is true. For each such row, connect the inputs, possibly negated by NOT, using AND to form a minterm; then connect all these rows using OR.
    2. Apply Product-of-Sums (POS, OR first, then AND) only to the rows where the output is false. For each such row, connect the inputs, possibly negated by NOT, using OR to form a maxterm; then connect all these rows using AND. (The two are equivalent formulations related by De Morgan’s laws.)

    For example, XOR:

    $$P\oplus Q \equiv (P\land eg Q)\ \lor\ ( eg P\land Q) \equiv ( eg P \lor eg Q)\land(P\lor Q) \equiv (P\lor Q)\land eg(P\land Q)$$

  • These two canonical forms also apply to multivariable functions with three or more inputs.



De Morgan’s Law of Equivalence

$$ eg(P\land Q)\ \equiv\ ( eg P)\ \lor\ ( eg Q)\qquad eg(P\lor Q)\ \equiv\ ( eg P)\ \land\ ( eg Q)$$

  • The negation of “a number is divisible by 3 or 5” is “it is divisible by neither 3 nor 5.”

  • The negation of “I am a boy” is not “I am female or I am an adult”; one must negate the statement carefully within the intended domain.

Conditional Statements


If

  • if P then Q: P→Q (P implies Q)
  • The proposition is false only when P is true and Q is false.
  • Therefore: “¬(P→Q) ≡ P ∧ ¬Q” → “P→Q ≡ ¬P ∨ Q”

Contrapositive

  • P→Q ≡ ¬Q→¬P
  • Negating without reversing is not equivalent.

Iff

  • P↔Q ≡ (P→Q) ∧ (Q→P)


Arguments

Modus Ponens: (P, P→Q) → Q Modus Tollens: (¬Q, P→Q) → ¬P


The “self-reference” fallacy

In Boolean logic, allowing propositions to assign truth values to themselves or to one another easily leads to paradox and destroys bivalence.

  • “P ≡ ¬P” makes P self-contradictory.

  • “P: Q ≡ c, Q: P ≡ t” makes P and Q mutually contradictory.

    Similar issues arise for sets: sets also cannot be allowed to refer to themselves or to one another arbitrarily.

  • Russell’s Paradox, which appears again in Lecture 2: $$W={,S\mid S\ ext{is a set and}\ S otin S,},\ W\ is\ self-contradictory$$

  • Barber’s Paradox: the barber shaves all and only those who do not shave themselves. Then does the barber shave himself? Either answer leads to contradiction.

    The core problem is naive comprehension: if one permits unconstrained self-reference or mutual reference of this kind, contradiction becomes unavoidable.
